# 컬럼 간 카드 이동 구현 플랜

작성일: 2026-03-03  
참조: UI/UX 설계서 5.2 컬럼 이동 모드, WebSocket CARD_MOVE 스펙

---

## 1. 목표

카드를 다른 컬럼으로 이동시키는 기능 구현.  
UI/UX 설계서 5.2에 정의된 "컬럼 이동 모드" 플로우를 따른다.

---

## 2. 기존 구현 상태

| 항목 | 상태 |
|------|------|
| WebSocket CARD_MOVE (to_column_id) | ✅ 지원 |
| REST API 카드 이동 | ✅ 지원 |
| 같은 컬럼 내 드래그 재정렬 | ✅ 구현됨 |
| 낙관적 업데이트 / CARD_MOVED | ✅ 구현됨 |
| 카드 우측 이동 아이콘 | ❌ 없음 |
| 컬럼 이동 모드 UI | ❌ 없음 |

---

## 3. 구현 단계

### Phase 1: CardTile에 이동 아이콘 추가
- [x] CardTile에 `onMove` 콜백 추가 (선택)

- [x] ReorderableListView의 드래그 핸들과 겹치지 않도록 레이아웃 조정

### Phase 2: 이동 모드 바텀시트 UI
- [x] `MoveCardSheet` 위젯 생성 (lib/widget/move_card_sheet.dart)
- [x] 진입: 선택된 카드 정보 표시, "이동 모드" 배지
- [x] 컬럼 목록: RadioListTile로 대상 컬럼 선택
- [x] 현재 카드가 속한 컬럼은 목록에서 제외
- [x] "이동 완료" / "취소" 버튼

### Phase 3: 이동 로직 연동
- [x] MoveCardSheet에서 대상 컬럼 선택 시 ws.moveCard 호출
- [x] toColumnId: 선택된 컬럼 ID
- [x] afterCardId: null (맨 위에 삽입)
- [x] 낙관적 업데이트: optimisticCardMoves에 반영
- [x] REST 폴백: updateCard(columnId, position: 0)
- [x] onRefresh 콜백으로 보드 상세 갱신

### Phase 4: 다국어 및 마무리
- [x] "이동 모드", "이동 완료", "moveCard" 등 번역 키 추가
- [x] 에러 처리 (ApiException)
- [x] 컬럼 1개일 때 이동 아이콘 미표시

## 4. UI/UX 설계서 요약 (5.2)

| 단계 | 설계 |
|------|------|
| 진입 | 카드 우측 이동 아이콘 탭 |
| 상태 | 상단 "이동 모드" 배지, 선택된 카드 강조, 좌우 스와이프로 컬럼 선택 |
| 완료 | "이동 완료" 버튼 → 선택된 컬럼 맨 위에 삽입 |
| 취소 | "취소" 버튼 또는 뒤로가기 |

---

## 5. 데이터 흐름

```
CardTile [이동 아이콘 탭]
  → showModalBottomSheet(MoveCardSheet)
  → 사용자가 대상 컬럼 선택 후 "이동 완료"
  → ws.moveCard(boardId, cardId, toColumnId, afterCardId: null)
  → optimisticCardMoves 업데이트
  → CARD_MOVED 수신 시 기존 로직으로 확정
  → onRefresh()
```
