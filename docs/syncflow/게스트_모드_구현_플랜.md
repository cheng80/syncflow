## 0. 합의된 정책 (고정)

| 항목 | 결정 |
|------|------|
| 게스트 로컬 보드 | **개수 제한 없음** |
| 진입 | **신규**: 첫 화면에서 **게스트로 시작** / **이메일로 로그인** 이원. **`has_ever_logged_in`**(또는 동등 플래그)이 있으면 **로그인·세션 복구 우선** (게스트 온보딩 강제 X). |
| 클라우드·협업 | **보드 참가**, **멤버 초대**, 그 외 세션 필수 API는 **계정 필요 다이얼로그** → 로그인. 가능하면 **`pending_intent`**로 로그인 후 원래 동작 재개. |
| 로그아웃 | **A안**: 세션·FCM 해제 후 **로그인 화면만** (게스트 모드로 자동 복귀 **하지 않음**). |
| 마이그레이션 | **전용 Import API** (`POST /v1/boards/import` 가칭). **단일 트랜잭션**, **`client_board_uuid` idempotency** (재요청 시 동일 결과 반환). |
| Import 실패 | **로컬 데이터 유지**, 재시도 안내. 서버는 **부분 반영 없음**(롤백). |
| Import 성공 | 서버 응답·목록 재조회 등으로 **확인 후** 로컬 해당 데이터 정리. |
| FCM | **게스트**: 토큰 **미등록** / 로그인 후 기존 플로우. |

## 1. 문서·오픈 API 초안

- [ ] Review Notes(앱 심사)용 한 단락: 게스트로 가능한 범위 vs 로그인 필요 시점

## 2. 백엔드 (FastAPI)

- [ ] `POST /v1/boards/import` 라우터 추가 (`X-Session-Token` 필수)
- [ ] 단일 DB 트랜잭션: `boards` + `columns` + `cards` 생성 (기존 스키마·제약 준수)
- [ ] 동일 `client_board_uuid` + 동일 `user_id` 재호출 시 **기존 보드 id·매핑 반환** (201 vs 200 정책 문서화)
- [ ] 소유자·멤버 레코드: import 시 `board_members`에 생성자 포함
- [ ] 멘션/WS: import 직후 **브로드캐스트 없음** 또는 정책 명시 (보통 단일 사용자면 생략 가능)
- [ ] 단위/통합 테스트: 성공·롤백·idempotent 재호출

## 9. 검증·배포 환경 (합의)

| 항목 | 결정 |
|------|------|
| **백엔드** | **게스트 모드 구현·사용성 검증 완료 전까지** 로컬 FastAPI만 사용. 검증 끝난 뒤 **NAS**에 실서버 배포. |
| **테스트 시 서버 기동** | **에이전트(개발 보조)가 주도적으로** 로컬 FastAPI 실행을 시도한다. 포트·의존성·권한 등으로 **불가능할 때만** 사용자에게 실행을 요청한다. |
| **클라이언트 검증 단말** | 시뮬레이터 **3대 고정**: **iPhone**, **iPad mini**, **Android 에뮬레이터** — 회귀·게스트 시나리오는 이 조합 기준으로 시나리오 작성. |
| **웹 빌드** | **테스트 대상에서 제외(폐기)** — Flutter Web / Chrome 등 E2E는 진행하지 않음. |

> QA 시나리오는 위 3단말 기준으로 작성한다. (필요 시 실기기는 NAS 배포 후 별도.)
