# 미니멀 협업 칸반 앱
## 계정 및 인증 아키텍처 설계서 (v1.0)

---

## 1. 프로젝트 목표

### 1.1 우선순위
1. 포트폴리오 설계력 증명
2. 빠른 마켓 등록
3. 장기 확장 가능성 확보

### 1.2 인증 설계 방향
- 디바이스 기반 ❌
- **User 기반 구조 채택**
- JWT는 사용하지 않음(초기 단계)
- UUID4 기반 세션 토큰 유지
- 이메일 6자리 인증 코드 방식 재사용

---

## 2. 전체 인증 구조 개요

- 이메일 입력
- 6자리 코드 발송
- 코드 검증
- user 생성/확인
- session_token(UUID4) 발급
- API / WebSocket 인증에 사용

---

## 3. 데이터베이스 설계

### 3.1 users
- id (PK, bigint)
- email (varchar, unique)
- email_verified_at (datetime)
- created_at (datetime)
- updated_at (datetime)

설명:
- 계정 중심 구조
- 이메일 기준 사용자 식별
- 추후 OAuth 확장 가능

### 3.2 email_verifications (기존 구조 유지)
- id
- email
- code_hash
- expires_at
- attempt_count
- created_at

정책:
- 6자리 코드 해시 저장
- 만료 5~10분
- 최대 시도 5회
- 재전송 제한 60초

### 3.3 sessions (신규)
- id (PK)
- user_id (FK)
- session_token (char 36, uuid4)
- expires_at (datetime)
- revoked (bool)
- created_at

정책:
- session_token = UUID4
- 만료 기간 = 14일
- 로그인 시 기존 세션 유지 허용
- 로그아웃 시 revoked = true
- 만료 시 자동 인증 실패

---

## 4. WebSocket 인증 전략

### 4.1 연결 방식
- `ws://server/ws?token=<session_token>`

### 4.2 서버 처리
1. session 조회
2. expires_at 확인
3. revoked 확인
4. user_id를 connection context에 저장

### 4.3 만료 처리
- 만료 시 서버에서 disconnect
- 클라이언트는 로그인 화면으로 전환

---

## 5. 협업 모델 설계

### 5.1 boards
- id
- owner_id (FK users.id)
- title
- created_at

### 5.2 board_members
- board_id
- user_id
- role (owner / member)
- joined_at

정책:
- owner만 초대 가능
- board 접근은 member만 가능
- WebSocket room join 시 권한 체크 필수

---

## 6. 계정 정책

### 6.1 계정 생성
- 이메일 인증 완료 시 자동 생성
- 기존 user 존재 시 로그인 처리

### 6.2 계정 삭제 (마켓 필수 대응)
- API: `DELETE /me`
- 동작:
  - user 삭제
  - sessions 삭제
  - board_members 삭제
  - owner 보드 삭제 또는 ownership 이전 정책 선택

---

## 7. 보안 정책
- 인증코드는 해시 저장
- session_token은 랜덤 UUID4
- HTTPS 필수
- WebSocket은 WSS 사용
- 로그인 시도 횟수 제한
- 세션 만료 14일

---

## 8. 확장 전략 (현재는 구현하지 않음)

향후 확장 가능 요소:
- device 테이블 분리
- JWT Access/Refresh 전환
- OAuth 로그인 추가
- 조직(Organization) 개념 도입
- Redis 세션 캐싱

현재는 구현하지 않음.

---

## 9. 클라이언트 설계 포인트 (Flutter)

### 9.1 로그인 흐름
- 이메일 입력
- 코드 입력
- session_token secure storage 저장
- 앱 시작 시 토큰 자동 검증 API 호출

### 9.2 자동 로그인
- 토큰 유효 → 홈 진입
- 만료 → 로그인 화면

### 9.3 로그아웃
- 세션 revoke API 호출
- secure storage 삭제
