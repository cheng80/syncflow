# 실시간 경량 협업 보드
## 멤버 초대 설계서 v1.1 (2차 적용)

본 문서는 **2차 확장 기능**으로 멤버 초대를 적용하기 위한 설계 문서이다.
(ERD에는 board_invites 스키마가 존재하더라도, 1차 MVP에서는 기능을 비활성화한다.)

---

# 1. 적용 시점

- 1차(MVP): 초대 기능 미적용
  - 소규모 모임/가족 사용은 운영으로 커버
- 2차: 초대 코드 기반 참여 기능 적용

---

# 2. 초대 방식 (2차 확정)

## 2.1 초대 코드 기반 참여

- 보드 오너가 초대 코드 생성
- 참여자는 앱 내에서 초대 코드 입력
- 서버 검증 후 board_members에 추가

딥링크/유니버설링크는 2차 이후 옵션.

---

# 3. 사용자 흐름

## 3.1 오너
1. 보드 진입
2. 설정/멤버 관리 진입
3. "초대 코드 생성" 버튼
4. 코드 표시 + 복사 + 공유

## 3.2 참여자
1. 보드 선택 화면
2. "보드 참가" 버튼
3. 초대 코드 입력
4. 성공 시 해당 보드 자동 추가

---

# 4. 데이터베이스

## 4.1 board_invites
- id (PK, bigint)
- board_id (FK → boards.id, not null)
- code (varchar(16), unique, not null)
- created_by (FK → users.id, not null)
- expires_at (datetime, not null)
- max_uses (int, default 20)
- used_count (int, default 0)
- revoked (boolean, default false)
- created_at (datetime, not null)

제약:
- expires_at 이후 사용 불가
- used_count >= max_uses 시 사용 불가
- revoked=true 시 사용 불가

---

# 5. API 정의 (REST)

## 5.1 초대 코드 생성
POST /boards/{board_id}/invites

권한: board owner

## 5.2 초대 코드 폐기
DELETE /invites/{invite_id}

권한: board owner

## 5.3 초대 코드로 참가
POST /invites/join

요청:
{ "code": "K7P3-9Q2A" }

처리:
- code 유효성/만료/폐기/횟수 검증
- 이미 멤버인지 확인(idempotent)
- board_members 추가
- used_count 증가

---

# 6. WebSocket 연동

참가 성공 시 보드 룸에 broadcast:
- MEMBER_JOINED

---

# 7. 보안 정책 (2차)

- 코드 길이 8~10자(대문자+숫자)
- 기본 만료: 7일
- 기본 max_uses: 20
- join API rate limit

---

- 보드 리스트: 보드 참가(코드 입력)
