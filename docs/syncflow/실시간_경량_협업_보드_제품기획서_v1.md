# 실시간 경량 협업 보드 앱
## 제품 기획서 (DB / 화면 설계 직전 단계)

---

# 1. 제품 정의

## 1.1 제품 한 줄 정의
소규모 팀을 위한 실시간 경량 협업 칸반 보드 앱.

## 1.2 핵심 가치
1. 실시간 상태 동기화
2. 템플릿 기반 빠른 시작
3. 미니멀하고 직관적인 사용성
4. 충돌 없이 안정적인 협업

---

# 2. 사용자 및 사용 시나리오

## 2.1 대상 사용자
- 2~5인 소규모 팀
- 콘텐츠 제작 팀
- 스터디 그룹
- 해커톤 팀
- 개인 프로젝트 협업 그룹

## 2.2 기본 사용 흐름
1. 회원 가입 / 로그인
2. 보드 생성 (템플릿 선택)
3. 멤버 초대
4. 카드 생성 / 이동 / 수정
5. 실시간 반영

---

# 3. 핵심 기능 범위 (MVP)

## 3.1 계정 및 인증
- 이메일 기반 6자리 코드 인증
- UUID4 세션 토큰
- 14일 세션 만료
- 로그아웃
- 계정 삭제

## 3.2 보드 기능
- 보드 생성 (템플릿 선택)
- 보드 수정 (제목 변경)
- 보드 삭제 (owner만)

## 3.3 템플릿 기능
- 기본 템플릿 제공 (3~5개)
- 템플릿은 JSON 구조
- 보드 생성 시 컬럼 자동 생성

## 3.4 컬럼 기능
- 컬럼 추가
- 컬럼 이름 수정
- 컬럼 삭제
- 컬럼 순서 변경

## 3.5 카드 기능
- 카드 생성
- 카드 제목 수정
- 카드 설명 수정 (경량 Markdown 지원)
- 카드 이동 (드래그)
- 우선순위 설정 (low / medium / high)
- 담당자 지정
- 마감일 설정

### 3.5.1 디스크립션(Markdown) 정책

MVP 허용 범위:
- 굵게 / 기울임
- 문단 / 줄바꿈
- 헤딩 (#, ##)
- 불릿/번호 리스트
- 링크

제외 범위:
- 코드 블록
- 인라인 코드
- 이미지
- HTML 태그

제한 정책:
- 최대 길이 제한 (예: 3000자)
- 리스트 화면에서는 Plain Text 2줄 프리뷰만 표시
- 상세 화면에서만 Markdown 렌더링

### 3.5.2 v1.1 확장 고려 (설계만 반영)
- 체크리스트 (- [ ], - [x])
- @멘션
- 카드 참조 링크 ([[CARD-ID]])

설계 점검 포인트:
- Markdown 파서 커스터마이징 가능 구조 유지
- 서버에서 Markdown 입력 검증 및 sanitize 처리
- custom inline 문법 확장 가능성 확보

## 3.6 협업 기능
- 실시간 카드 이동 반영
- 실시간 카드 수정 반영
- 보드 접속 사용자 표시 (presence)
- 카드 편집 중 soft lock 표시 (MVP 포함)

### 3.6.1 Soft lock 정책 (MVP)

목표: 동시 편집 충돌을 최소 비용으로 예방.

- 잠금 단위: 카드 단위
- 잠금 타입: soft lock (TTL 기반)
- TTL: 30초 / 편집 중 renew
- 저장/취소 시 즉시 해제
- TTL 만료 시 자동 해제
- WebSocket disconnect 시 해제

클라이언트 UX:
- 잠긴 카드 상세는 읽기 전용
- 잠근 사용자 표시
- “편집 중…” 라벨 표시

# 4. 실시간 동기화 정책

## 4.1 통신 방식
- WebSocket 기반 양방향 통신
- 연결 시 세션 토큰 검증

## 4.2 서버 권위 모델
- 서버가 최종 상태 결정
- 클라이언트는 optimistic update 사용
- 서버 응답 기준으로 상태 확정

## 4.3 충돌 처리 정책
- 카드 편집 시 soft lock 적용
- 이동 충돌 발생 시 서버 상태 우선

---

# 5. 권한 정책

## 5.1 역할 정의
- owner
- member

## 5.2 권한 범위

Owner:
- 보드 삭제
- 멤버 초대/제거
- 컬럼 구조 수정

Member:
- 카드 생성/수정/이동
