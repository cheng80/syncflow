# 3대 이동 동기화 불일치 원인 분석

> 증상: 카드 내용은 3대 동기화, 이동 데이터는 2대만 됨. 카드 정렬 위치가 3대 다 달라지는 때가 있음.

---

## 1. 원인: board_version 기반 구버전 이벤트 무시

### 현재 로직 (문제)

```dart
// 모든 카드 이벤트에 공통 적용
if (incomingVersion != null && currentVersion != null && incomingVersion < currentVersion) {
  return;  // 구버전 이벤트 전체 무시
}
```

- `board_version`은 **보드 전체** 단일 값
- `CARD_MOVED`는 **해당 컬럼만** 갱신 (`column_cards`)

### 문제 시나리오

| 시간 | 기기 A | 기기 B | 기기 C |
|------|--------|--------|--------|
| T1 | 컬럼1 카드 이동 → CARD_MOVED(v=1001) | - | - |
| T2 | - | 컬럼2 카드 이동 → CARD_MOVED(v=1002) | - |
| T3 | - | - | B 수신 → 적용, currentVersion=1002 |
| T4 | - | - | A 수신 → **1001 < 1002 → 무시** |

**결과**: C는 컬럼1 이동을 반영하지 못함. 컬럼별로 다른 이벤트인데, 전역 버전 때문에 무시됨.

### 네트워크 순서 뒤바뀜

- 서버는 A → B 순으로 처리 (v1001, v1002)
- C는 B → A 순으로 수신 가능
- C가 B를 먼저 적용하면 A를 구버전으로 잘못 무시

---

## 2. 해결 방향

### 옵션 A: CARD_MOVED는 board_version 체크 제외 (간단)

- `CARD_MOVED`만 버전 체크를 건너뛰고 항상 적용
- **위험**: 같은 컬럼에 대한 이벤트가 B→A 순으로 오면, A가 B를 덮어써 잘못된 상태가 됨

### 옵션 B: 컬럼별 버전 추적 (권장)

- `lastAppliedColumnVersion[columnId] = version` 유지
- `CARD_MOVED` 수신 시:
  - `incomingVersion > lastAppliedColumnVersion[columnId]` 이면 적용
  - 그렇지 않으면 무시 (해당 컬럼만)
- 다른 컬럼 이벤트는 서로 영향 없음

### 옵션 C: CARD_MOVED는 항상 적용 (실용적 타협)

- CARD_MOVED는 버전 체크 없이 적용
- 같은 컬럼에서 B→A 순 수신 시 잘못된 상태 가능성은 있으나, 3대 동시 사용 시 빈도는 낮음
- 우선 적용 후, 문제 재현 시 옵션 B로 전환

---

## 3. 적용한 수정 (단순화)

- **refetch 스킵만 유지**: 캐시가 더 최신이면 덮어쓰지 않음
- 컬럼별 버전 추적은 제거 (복잡도 감소)

---

## 4. 추가 원인: refetch가 CARD_MOVED보다 오래된 캐시로 덮어씀

### 시나리오

1. 기기 A가 카드 이동 → CARD_MOVED 브로드캐스트
2. 기기 B가 CARD_MOVED 수신 → 캐시 반영 (boardVersion 갱신)
3. 기기 B가 onRefresh/invalidate로 refetch 트리거 (이동 직후 등)
4. refetch 응답이 CARD_MOVED 커밋 **이전** 시점의 스냅샷일 수 있음 (레이스)
5. ref.listen에서 캐시를 refetch 결과로 **덮어씀** → CARD_MOVED 반영분 손실
6. 점차 각 기기 캐시가 달라짐

### 수정: 캐시가 더 최신이면 덮어쓰지 않음

```dart
// ref.listen 내부
final current = ref.read(boardDetailCacheProvider(widget.boardId));
final newVersion = detail.boardVersion ?? 0;
final currentVersion = current?.boardVersion ?? 0;
if (newVersion < currentVersion) {
  return; // CARD_MOVED로 이미 더 최신 상태 → 덮어쓰지 않음
}
```

---

## 5. 서버 board_version 정밀도 (추가 검토)

- `boards.updated_at`은 DATETIME(초 단위)
- 같은 초 내 여러 이동 시 동일 board_version 가능 → 순서 판단 오류
- 개선 후보: `sync_version` 컬럼 추가 (현재 미적용, 복잡도 고려)

---

## 6. 추가 확인 사항

- **브로드캐스트**: `broadcast_to_board`가 3대 모두에 전송하는지
- **WebSocket 연결**: 3대 모두 동일 보드 룸에 JOIN 되어 있는지
- **수신 로그**: `[동기화] 구버전 이벤트 무시` 로그가 찍히는지
