# NEW FCM 구현 및 설정 가이드 (Riverpod 기본)

이 폴더는 Flutter + FCM을 **Riverpod 기반으로 표준화**해서 빠르게 적용하기 위한 가이드 모음입니다.
모든 본문 문서는 Riverpod 기준 세팅을 기본값으로 가정합니다.

문서 상태:
- 신규 사용 경로: `docs/NEW-FCM구현 및 설정가이드`
- 기존 문서 보관 경로: `docs/_archive_FCM구현 및 설정 가이드`

검증 기준:
- 공식 문서 반영일: **2026-03-05**
- 기준 소스: Firebase/Google 공식 문서 + Firebase Flutter 공식 패키지 문서

## 문서 구성

1. `01_FCM_iOS_Android_설정_가이드.md`
- iOS/Android FCM 기본 세팅
- Riverpod 기반 앱 부팅/FCM 초기화 구조
- 포그라운드 수신(`onMessage`) + `flutter_local_notifications` 표준

2. `02_FCM_토큰_저장소_및_서버_연동_가이드.md`
- 토큰 수명주기(발급/갱신/비활성화) 표준
- Riverpod + Secure Storage/GetStorage 저장 전략
- 서버 업서트/재시도 기준

3. `03_FCM_백엔드_푸시_발송_가이드.md`
- 이벤트 기반 푸시 발송 파이프라인
- HTTP v1/Admin SDK 기준 발송
- 무효 토큰 정리/운영 지표

4. `04_FCM_푸시_시스템_다이어그램.md`
- 시스템 흐름/컴포넌트/데이터 흐름 템플릿

## 기본 아키텍처 원칙

- 상태 관리는 Riverpod을 기본으로 사용
- FCM 로직은 `main.dart`에 누적하지 않고 `Notifier`로 분리
- 민감값(세션, 필요 시 FCM 토큰)은 Secure Storage 우선
- 비민감 메타(마지막 동기화 시각 등)는 GetStorage 사용 가능

## 비-Riverpod 프로젝트

- 이 문서는 Riverpod을 기본 전제로 설명한다.
- Riverpod 미사용 프로젝트는 동일한 흐름(초기화 -> 권한 -> 토큰 업서트 -> 갱신 리스너 -> 포그라운드 표시)을 해당 상태관리 방식으로 매핑해 적용한다.

## 권장 도입 순서

1. `01` 앱 FCM 토큰 발급/권한/포그라운드 표시 성공
2. `02` 토큰 서버 등록/갱신/비활성화 완료
3. `03` 이벤트 기반 푸시 발송 구현
4. `04` 다이어그램/운영 기준 팀 합의

## 공통 완료 기준

- 토큰 발급/등록/갱신이 자동 동작한다.
- 포그라운드/백그라운드 모두 사용자 가시 알림이 검증된다.
- 로그아웃/탈퇴 시 토큰 비활성화 정책이 지켜진다.
- 운영 지표(성공률/무효 토큰률/지연)가 수집된다.
